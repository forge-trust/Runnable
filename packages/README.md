# Runnable v0.1 package chooser

> Generated from `packages/package-index.yml` and evaluated project metadata. Do not edit this file by hand.

Runnable v0.1 is a coordinated .NET 10 package family. Start with the package that matches the app you're building, then add optional modules only when your app needs them.

All direct-install packages currently target `net10.0`. Install commands below use `dotnet package add`; if you are on an older SDK, use `dotnet add package` instead.

## Web app

Install this first for a normal ASP.NET Core app with Runnable modules when you want the base web startup pipeline before layering on optional UI or docs packages.

```bash
dotnet package add ForgeTrust.Runnable.Web
```

What you get: Base ASP.NET Core hosting, middleware and endpoint composition, and the core Runnable web module contract.

Not included: OpenAPI generation, Scalar UI, RazorWire reactive UI, Tailwind build integration, or the RazorDocs host.

## Also building...

- Start with `ForgeTrust.Runnable.Console` for CLI commands or worker-style processes that should share Runnable module composition.
- Add `ForgeTrust.Runnable.Dependency.Autofac` when your app already uses Autofac and your modules need Autofac-specific registrations.
- Start with `ForgeTrust.Runnable.Aspire` for distributed app-model projects and service-default composition.
- Add `ForgeTrust.Runnable.Web.Scalar` after `ForgeTrust.Runnable.Web.OpenApi` when you want a hosted API reference UI.
- Add `ForgeTrust.Runnable.Web.RazorWire` when you want reactive Razor UI, Turbo-style streams, and server-rendered islands.
- Add `ForgeTrust.Runnable.Web.Tailwind` when you want Tailwind without a separate Node.js asset pipeline.

## Support and proof-host surfaces

### Support and runtime packages

- `ForgeTrust.Runnable.Web.Tailwind.Runtime.linux-arm64`: Tailwind standalone runtime for Linux Arm64 build hosts. Restore it transitively through `ForgeTrust.Runnable.Web.Tailwind` instead of installing it directly.
- `ForgeTrust.Runnable.Web.Tailwind.Runtime.linux-x64`: Tailwind standalone runtime for Linux x64 build hosts. Restore it transitively through `ForgeTrust.Runnable.Web.Tailwind` instead of installing it directly.
- `ForgeTrust.Runnable.Web.Tailwind.Runtime.osx-arm64`: Tailwind standalone runtime for macOS Arm64 build hosts. Restore it transitively through `ForgeTrust.Runnable.Web.Tailwind` instead of installing it directly.
- `ForgeTrust.Runnable.Web.Tailwind.Runtime.osx-x64`: Tailwind standalone runtime for macOS x64 build hosts. Restore it transitively through `ForgeTrust.Runnable.Web.Tailwind` instead of installing it directly.
- `ForgeTrust.Runnable.Web.Tailwind.Runtime.win-x64`: Tailwind standalone runtime for Windows x64 build hosts. Restore it transitively through `ForgeTrust.Runnable.Web.Tailwind` instead of installing it directly.

### Docs and proof hosts

- `ForgeTrust.Runnable.Web.RazorDocs`: Reusable docs package for harvesting repository docs into a RazorDocs UI. This is a real package, but it is a proof-host surface rather than the default first install for general Runnable apps. Start here: [RazorDocs README](../Web/ForgeTrust.Runnable.Web.RazorDocs/README.md)
- `ForgeTrust.Runnable.Web.RazorDocs.Standalone`: Thin runnable host for serving or exporting RazorDocs. Treat it as a proof host and example app, not a package you install into another project. Start here: [Standalone host README](../Web/ForgeTrust.Runnable.Web.RazorDocs.Standalone/README.md)

### Not in the direct-install matrix

- `ForgeTrust.Runnable.Web.RazorWire.Cli`: Held out of the direct-install chooser until issue #171 lands real .NET tool packaging and stable install guidance.
